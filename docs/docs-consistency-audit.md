# Docs Consistency Audit

Date: 2026-06-25.

Scope: `README.md`, `AGENTS.md`, `docs/design/goal-runner.md`, and `docs/developer/index.md`.

This audit compares documented claims against source files and records whether each claim is accurate, fixed, generated, or needs human input.

## README.md

| Claim | Evidence | Status |
| --- | --- | --- |
| Neo is a Go CLI/TUI project using Bubble Tea. | `go.mod` requires `charm.land/bubbletea/v2`; `cmd/neo/main.go` dispatches the CLI; `internal/tui/model.go` owns the Bubble Tea model. | Accurate. |
| `neo` and `neo chat` open interactive chat. | `cmd/neo/main.go` routes no args and `chat` to `runChat`. | Accurate. |
| Supported shell commands include `neo sessions`, `neo resume <id>`, `neo login`, `neo logout`, and `neo help`. | `cmd/neo/main.go` dispatches each command and `printUsage` lists them. | Accurate. |
| Config resolution is `./neo.yaml`, then `~/.neo/config.yaml`, then embedded defaults. | `internal/config/config.go` implements that order in `Load`. | Accurate. |
| The default provider is Anthropic. | `internal/config/config.go` sets `defaultProvider = "anthropic"` and `internal/config/defaults/neo.yaml` uses `provider: anthropic`. | Accurate. |
| OpenAI supports API-key and subscription auth. | `internal/config/config.go` defines `api_key` and `subscription`; `cmd/neo/main.go` chooses `openai.New` or `newCodexProvider`. | Accurate. |
| OpenRouter is a supported provider. | `cmd/neo/main.go` accepts `provider: openrouter`; `internal/config/defaults/neo.yaml` documents `OPENROUTER_API_KEY`; `internal/llm/openrouter/client.go` implements the adapter. | Fixed in README. |
| `permissions.mode` defaults to `trusted`. | `internal/config/config.go` defaults empty permissions mode to `trusted`; `internal/config/defaults/neo.yaml` sets `mode: trusted`; generated docs say the same. | Fixed in README. |
| Permission modes are `ask`, `trusted`, and `readonly`. | `internal/config/config.go` validates only those three values; `internal/permission/policy.go` implements their decisions. | Accurate. |
| Path-shaped file and search tools are workspace-bound. | `internal/permission/policy.go` checks `read_file`, `write_file`, `edit_file`, `grep`, and `glob` paths with `workspace.ResolveWithin`. | Accurate. |
| The chat tool registry includes file, search, bash, workflow, and subagent tools. | `cmd/neo/main.go` registers `workflow.Tool` and `factory.AgentTool`; `newRegistry` registers `bash`, `read_file`, `write_file`, `edit_file`, `grep`, and `glob`. | Fixed in README. |
| `AGENTS.md` loading is feature-flagged and reads global plus project docs. | `cmd/neo/main.go` calls `projectctx.Load` when `AgentsFileEnabled`; `internal/projectctx/projectctx.go` reads `~/.neo/AGENTS.md` and repo ancestors. | Accurate. |
| Skills live in `~/.neo/skills` and project `.neo/skills`, and `$name` expands a skill body. | `internal/skills/skills.go` discovers both locations and implements `Expand`; `.neo/skills/commit`, `.neo/skills/coordinator-worker`, and `.neo/skills/review` are tracked examples. | Accurate. |
| Sessions are stored under `~/.neo/sessions` and can be resumed from shell or the TUI. | `internal/session/session.go` defines `DefaultDir`; `cmd/neo/main.go` implements shell resume; `internal/tui/sessions.go` implements the in-TUI browser. | Accurate. |
| `/memory <text>` appends project memory. | `internal/tui/info.go` registers `/memory` when enabled; `internal/projectctx/memory.go` appends `memory.md` at the project root. | Accurate. |
| Releases build Linux and macOS `amd64` and `arm64` archives and update the Homebrew cask. | `.github/workflows/release.yml` runs GoReleaser on `v*` tags; `.goreleaser.yaml` lists linux/darwin and amd64/arm64 plus `homebrew_casks`. | Accurate. |

## docs/design/goal-runner.md

| Claim | Evidence | Status |
| --- | --- | --- |
| `/goal` is a design proposal, not current implemented behavior. | The document says `Status: Draft`; `rg` finds no `internal/goal` package and no `/goal` slash command in `internal/tui/info.go` or `internal/tui/model.go`. | Accurate. |
| Current Neo has a cancellable send path. | `internal/tui/model.go` stores `sendCancel`, Esc calls it while busy, and `startSend` passes a cancellable context into `Agent.SendWith`. | Accurate. |
| The agent passes cancellation through provider and tool calls. | `internal/agent/agent.go` passes `ctx` into `Provider.Complete` and `runTool`; `runTool` passes it into tool `Run`. | Accurate. |
| Bash cancellation reaches the process. | `internal/tools/bash.go` uses `exec.CommandContext`. | Accurate. |
| File and search tools observe cancellation only in some paths. | `internal/tools/fs.go` checks context in line-window reads; `internal/tools/search.go` checks context during `glob` walks and after `grep` file discovery begins, but `Grep.Run` calls `filesUnder(target)` before its first `ctx.Err()` check. | Fixed in audit. `readWholeFileCapped`, `write_file`, `edit_file`, and grep's initial file discovery are not interrupt-aware today. |
| Existing TUI chat input is suppressed while a turn is busy. | `internal/tui/model.go` ignores normal chat text when `m.busy` is true; slash commands are parsed before the busy gate and some require idle. | Accurate. |
| Workflow is separate from goal lifecycle control. | `internal/workflow/workflow.go` only manages checklist state; there is no `goal` tool today. | Accurate as a design boundary. |
| External Hermes, Claude Code, and Codex source summaries are correct. | The file links external sources, but this audit was scoped to the local Neo repository and did not re-verify external projects. | Needs human input if external-source accuracy must be certified. |
